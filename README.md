# retail-ai-sop-assistant

![Project Cover](./assets/project-cover.svg)

> 一个面向零售商品管理场景的 AI 知识库 / ERP 操作 SOP 助手示例项目。

本项目基于通用零售业务流程整理 SOP 文档与 FAQ，用于模拟企业内部新人在商品资料维护、库存异常排查、调拨处理、销售数据核对等场景下，通过自然语言查询处理步骤、注意事项和风险提醒。

## 项目简介

本项目是一个基于零售商品管理场景的 AI 知识库 / ERP 操作 SOP 助手，主要用于模拟企业内部员工在处理商品资料、库存、调拨、销售数据等业务问题时，通过 AI 问答方式快速查询操作规范和排查思路。

项目不使用任何真实企业数据，所有内容均基于通用零售业务流程整理，适合作为 AI 应用实施、知识库搭建、业务流程梳理和自动化工具落地的练习项目。

## 项目背景

在零售企业中，商品管理相关工作通常涉及 ERP 系统、商品资料、SKU、门店库存、区域调拨、销售数据核对等流程。新人在刚接触业务系统时，容易遇到以下问题：

- 不清楚 ERP 操作前需要确认哪些信息；
- 不知道库存异常应该从哪些维度排查；
- 对调拨单、销售数据、商品资料之间的关系理解不清；
- 遇到数据不一致时，缺少标准化排查步骤；
- 业务经验依赖人工口头传递，难以沉淀复用。

因此，本项目尝试将常见业务流程整理为 SOP 文档，并通过 AI 知识库问答的方式，让用户可以用自然语言查询处理流程和注意事项。

## 主要功能

- 商品资料维护 SOP 查询
- 库存异常排查流程查询
- 调拨单处理流程查询
- 销售数据核对思路查询
- ERP 操作风险提醒
- 新员工常见问题问答
- 基于 Prompt 的结构化回答输出
- 静态演示页展示项目结构和回答示例

## 推荐回答格式

AI 助手回答时建议固定输出以下结构：

```text
问题判断：
处理步骤：
需要确认的信息：
风险提醒：
建议后续动作：
```

## Dify / Coze 搭建建议

1. 创建一个新的知识库问答应用或 Bot。
2. 上传 `docs/` 下的 6 个 Markdown 文档。
3. 将 `prompts/assistant_prompt.md` 或 `07_知识库系统提示词模板.md` 作为系统提示词基础。
4. 设置回复必须固定输出 5 个模块。
5. 使用 [08_知识库测试问题集.md](./08_知识库测试问题集.md) 做上线前验收。

## 项目价值

通过该项目，可以练习以下能力：

- 将真实业务场景拆解为标准化流程；
- 将零散经验整理为知识库文档；
- 使用 AI 工具搭建问答助手；
- 设计结构化 Prompt，提高回答稳定性；
- 模拟 AI 应用实施中的需求理解、知识整理、测试验证和迭代过程。
