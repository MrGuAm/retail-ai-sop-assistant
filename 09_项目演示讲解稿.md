# 项目演示讲解稿

## 1. 使用场景
这份文档适合你在以下场景直接使用：
- 面试时介绍项目
- GitHub 仓库视频录屏配音
- 简历项目追问时的口头说明
- 向同学、老师或面试官快速展示成果

## 2. 30 秒版本

```text
我做了一个“零售业务 AI 知识库 / ERP 操作 SOP 助手”项目，主要解决零售公司新人或商品助理不熟悉 ERP 操作的问题。我先整理了商品资料维护、库存异常排查、调拨单处理、销售数据核对这些通用 SOP，再补了 20 个高频 FAQ，并设计了固定回答模板，让 AI 每次都输出问题判断、处理步骤、需要确认的信息、风险提醒和建议后续动作。这样这个项目既能展示业务理解，也能体现我用 Dify 或 Coze 搭知识库应用的能力。
```

## 3. 60 秒版本

```text
这个项目是一个模拟企业内部使用场景的 AI 助手，名字叫“零售业务 AI 知识库 / ERP 操作 SOP 助手”。它的目标不是做很复杂的模型，而是把零售业务里常见但容易出错的流程整理出来，帮助新人更快上手。

我主要做了三部分内容。第一部分是业务知识整理，我写了 5 篇 SOP，覆盖商品资料维护、库存异常排查、调拨单处理、销售数据核对和 ERP 操作风险清单。第二部分是知识库问答设计，我整理了 20 个高频问题，并约束 AI 固定输出 5 个模块，保证答复结构统一、可执行、可追溯。第三部分是项目展示，我补了一个静态演示页、系统提示词模板和测试问题集，让这个项目既能放 GitHub，也方便后面接到 Dify 或 Coze 上做演示。
```

## 4. 2 分钟版本

```text
我做这个项目的出发点是，很多 ERP 或零售运营问题并不是技术太难，而是新人不知道先查什么、按什么顺序查、哪些动作不能直接做。所以我想做一个更偏业务落地的 AI 知识库助手。

在内容上，我先按照零售业务里最常见的几个环节整理资料，包括商品资料维护、库存异常排查、调拨单处理、销售数据核对和 ERP 操作风险清单。每份文档都不是只写定义，而是尽量写成“可以照着执行”的 SOP，比如处理前先确认什么、按什么顺序排查、哪些情况属于高风险、处理完之后还要复核什么。

在问答设计上，我没有让 AI 自由发挥，而是做了固定模板，要求它每次都输出问题判断、处理步骤、需要确认的信息、风险提醒和建议后续动作。这样做的好处是，新员工拿到回答后更容易执行，也能减少 AI 给出模糊或跳步建议的情况。

在展示层面，我还补了一个静态演示页，方便放到 GitHub Pages 或录屏展示；另外还准备了系统提示词模板和测试问题集，方便在 Dify、Coze 这类平台里快速搭一个可运行的知识库 Demo。

如果继续扩展，我会往两个方向做：一个是补更多 SOP，比如门店补货、采购收货和盘点；另一个是接一个简单前端或聊天界面，让它更接近真实企业内部助手产品。
```

## 5. 面试官可能追问的问题

### 这个项目的难点是什么？
建议回答：

```text
难点不是代码实现，而是把业务知识整理成既适合知识库检索、又适合新人执行的结构化内容。比如我需要考虑文档怎么切分、FAQ 怎么覆盖高频场景、提示词怎么约束回答格式，避免 AI 回答得太空泛。
```

### 这个项目体现了你什么能力？
建议回答：

```text
我觉得主要体现了三点：第一是业务理解和流程抽象能力；第二是把业务资料转成知识库内容的能力；第三是产品化思维，比如我不仅整理文档，还考虑了回答格式、测试问题集和演示方式。
```

### 如果真的在企业里落地，你会怎么继续做？
建议回答：

```text
我会先补真实场景下更细的 SOP 和角色权限说明，再接企业微信、飞书或网页聊天界面，之后根据提问日志继续补 FAQ 和知识库召回效果。另外我也会增加反馈机制，记录哪些问题 AI 回答得不准，再持续优化文档和提示词。
```
