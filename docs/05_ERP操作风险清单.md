# ERP 操作风险清单

## 1. 文档目的
整理零售业务中常见的 ERP 操作风险，帮助新人在录单、改资料、查库存、做调拨和核对报表时减少低级错误。

## 2. 使用方式
- 每次操作前先对照对应风险项
- 发现高风险动作时先暂停，再确认
- 无审批、无证据、无复核的操作一律谨慎处理

## 3. 风险分类与控制建议

### 3.1 商品主数据风险
**风险场景**
- 重复建商品
- 条码录错
- 分类、税率、单位维护错误

**可能影响**
- 销售识别错误
- 库存混乱
- 报表口径失真
- 财务核算异常

**控制建议**
- 先查重，再建档
- 条码录入后做唯一性校验
- 关键字段至少二次复核

### 3.2 价格维护风险
**风险场景**
- 零售价录错小数位
- 活动价维护到标准价字段
- 生效日期设置错误

**可能影响**
- 门店错价销售
- 毛利异常
- 顾客投诉

**控制建议**
- 价格维护后抽查 POS 展示
- 涉及促销的价格调整必须区分字段
- 批量改价前先在测试商品上验证

### 3.3 调拨操作风险
**风险场景**
- 未审核先出库
- 只做出库不做入库
- 调错门店或仓库

**可能影响**
- 在途库存长期挂账
- 门店缺货或虚增库存
- 责任归属不清

**控制建议**
- 按申请、审核、出库、入库顺序操作
- 每日检查未完结调拨单
- 调拨前确认组织和仓库编码

### 3.4 库存调整风险
**风险场景**
- 未查原因直接调账
- 盘点差异未经复核就审核
- 手工报损数量随意录入

**可能影响**
- 掩盖真实问题
- 账实差异扩大
- 经营数据失真

**控制建议**
- 调账前保留异常证据
- 盘点差异必须有复盘记录
- 库存调整要有审批和备注

### 3.5 销售数据风险
**风险场景**
- 报表口径理解错误
- 把退货当正向销售统计
- 跨天数据未同步完成就出报表

**可能影响**
- 销售分析错误
- 经营决策失真
- 与财务口径冲突

**控制建议**
- 出报表前先确认数据口径
- 明确销售额、净销售额、含税与未税定义
- 跨系统核对时统一统计时间

### 3.6 权限与审批风险
**风险场景**
- 新员工拥有过高权限
- 共用账号操作
- 审批流被跳过

**可能影响**
- 无法追责
- 高风险操作无人把关
- 数据被误改

**控制建议**
- 权限按岗位最小化分配
- 禁止共用账号
- 审批记录必须可追溯

### 3.7 时间与批次风险
**风险场景**
- 单据日期录错
- 跨月单据回填错误期间
- 食品类商品批次、效期漏填

**可能影响**
- 月结异常
- 库存先后顺序错乱
- 临期品管理失控

**控制建议**
- 营业结束后谨慎补录跨天单据
- 涉及月结的改单必须与财务同步
- 批次商品必须强制校验效期

### 3.8 数据导出与传递风险
**风险场景**
- 手工二次整理报表时覆盖原数据
- 将旧版本文件当作最新版本使用
- 通过聊天工具转发截图代替正式数据

**可能影响**
- 数据引用错误
- 版本混乱
- 后续无法追溯

**控制建议**
- 原始导出文件单独保存
- 文件命名包含日期和版本号
- 重要结论以表格或正式记录为准

## 4. 新员工高频注意事项
- 先查、再做、后提交，不要边猜边操作。
- 不清楚口径时先问清楚再导报表。
- 看见异常先留证据，不要先把数据改掉。
- 同一个问题不要同时找多人重复处理，先确认责任人。
- 任何影响库存、价格、财务口径的修改，都要留下说明。
